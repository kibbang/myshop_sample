# MyShop Admin (Spring MVC + JPA + Thymeleaf)

쇼핑몰 운영을 위한 **관리자(Back Office)** 웹 애플리케이션입니다.  
**Spring MVC + Spring Data JPA + Thymeleaf** 기반으로, 상품 관리 중심 기능부터 점진적으로 확장하는 형태로 개발 중입니다.

- 진행 상태: **현재 진행중(기능 확장 단계)**
- DB: **MySQL**
- 인증: **관리자 로그인/권한 흐름 적용(동작)**

---

## 1) 프로젝트 목표

- 운영자 관점에서 필요한 기능(상품/계정 등)을 **관리자 화면**으로 제공
- Controller/Service/Repository/Domain 계층 분리로 **유지보수성** 확보
- 화면 입력 DTO와 서비스 DTO를 구분하여 **변경 영향 최소화**
- 공통 인증/인가 및 웹 계층 공통 처리(인터셉터/리졸버 등)로 **확장 가능한 구조** 지향

---

## 2) 기술 스택

- **Java 21**
- **Spring MVC**
- **Spring Data JPA (Hibernate)**
- **Jakarta EE (`jakarta.*`)**
- **Thymeleaf**
- **MySQL**
- **Gradle**
- **Lombok**

---

## 3) 기능 요약

### 관리자(Admin)
- 관리자 홈/메뉴 기반 레이아웃
- 상품 관리
  - 상품 목록 화면
  - 상품 등록 화면
  - 상품 등록 처리(유효성 검증 포함)
  - 판매 상태/통화 등 Enum 기반 관리

### 인증(Auth)
- 관리자 인증이 적용되어 있으며(로그인/권한), 보호된 관리자 기능 접근 제어를 전제로 설계

> 향후 확장: 상품 수정/삭제, 페이징/검색/정렬, 카테고리/재고/옵션, 이미지 업로드, 감사 로그 등

---

## 4) 화면 구성(Thymeleaf)

- 공통 관리자 레이아웃(사이드바/상단/컨텐츠 영역)을 사용
- 각 기능 화면은 “컨텐츠 프래그먼트”를 교체하는 방식으로 구성하여
  - 페이지 중복을 줄이고
  - 메뉴 활성화/레이아웃 유지가 쉬운 구조를 지향합니다.

---

## 5) 포트폴리오 관점 “구현 포인트”

- **계층형 아키텍처**: Web(Controller) / Service / Repository / Domain 분리
- **DTO 분리 전략**: 화면 입력 모델과 서비스 입력 모델을 분리해 의존성/결합도 감소
- **Validation & 에러 흐름**: 입력 검증 → 오류 시 동일 화면 재노출 → 정상 시 리다이렉트 흐름
- **인증/인가 적용**: 관리자 기능 보호를 전제로 기능 설계
- **템플릿 레이아웃 전략**: Thymeleaf 레이아웃 + 프래그먼트
